# 美术资源清单（占位图已生成 → 待美术替换）

> **现状**：已生成 36 张「白底 + 顶部分类色条 + 居中中文名」占位 PNG，全部就位于
> `packages/client/public/art/`，游戏 UI 已接入（`client/src/info.ts` 的 icon 字段
> 已改为 `/art/*.png` 路径，渲染走 `main.ts` 的 `art()` 助手，加载失败回退文字徽标）。
>
> **替换方式**：美术出图后，按**相同文件名**覆盖 `packages/client/public/art/` 下的同名
> 文件即可，**无需改任何代码**。占位图由 `tools/gen_art.py`（Pillow）生成，文件名与中文
> 标签的唯一事实源就在该脚本的 `ASSETS` 表里。
>
> **统一规范**：
> - 格式 PNG，图标类 128×128（占位图当前为白底，正式美术建议透明背景）。
> - 命名：`类别_名称.png`，全小写下划线，如 `res_wood.png`、`unit_legionnaire.png`。
> - 风格基调：罗马/古典战争策略，暖色木质 UI，写实偏插画。可后续在此统一指定色板。
> - 占位图分类色条（仅用于占位区分，非最终配色）：资源=棕 / 资源田=绿 / 建筑=蓝 /
>   兵种=红 / 野怪=紫 / 界面=黄褐。

## 替换流程（美术出图后）

1. 把图片放进 `packages/client/public/art/`，**按上面的文件名覆盖**同名占位图。
2. 重新构建即可生效（`npm run build`），**无需改 `info.ts` 或任何前端/后端代码**。
   - 渲染统一走 `main.ts` 的 `art()` 助手输出 `<img>`，图片加载失败会自动回退为文字徽标，
     所以即使个别图缺失，界面也不会破版。
3. 想批量重生成占位图（例如新增了兵种/建筑）：编辑 `tools/gen_art.py` 的 `ASSETS` 表，
   运行 `python tools/gen_art.py`。
