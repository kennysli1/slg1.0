# 美术生成规范（生成就绪 · 可直接投喂图像模型）

> 本文件是给「图像生成模型」直接用的出图 spec。视觉基调：**通用中世纪 / 奇幻策略游戏**
> （不绑罗马、不绑任何既有品牌）。文件名 / 中文名以 `packages/client/src/info.ts`
> 与 `tools/gen_art.py` 为唯一事实源，本文件与之对齐。
>
> **用法**：先把下面「一、统一风格块」整段作为 system / style 前缀，再逐张接上「三、逐图 prompt」
> 里对应那一行的 subject 提示词。**同一类别（如 10 个建筑）建议成批生成**，共用同一风格块，
> 才能保证成套一致。出图后按**相同文件名**覆盖 `packages/client/public/art/`，无需改代码。

## 一、统一风格块（所有图共用，放在每条 prompt 前面）

```
Game asset icon for a medieval-fantasy strategy game. Hand-painted, stylized
semi-realistic illustration. Warm earthy medieval palette: aged wood browns,
weathered stone grey, muted forest green, dark iron, parchment cream, with
gold and deep-red heraldic accents. Soft warm light from the upper-left,
gentle ambient occlusion, clean readable silhouette, subtle painterly texture.
Single centered subject with even padding, no ground plane unless specified.
NEGATIVE: no text, no letters, no watermark, no signature, no border/frame,
no drop shadow outside the subject, no photo-realism, no modern objects,
no busy background.
```

## 二、技术规格（硬性，喂模型时明确写出）

| 项 | 要求 |
|---|---|
| 生成尺寸 | **1024×1024**，正方形；出图后统一缩放到 128×128 使用 |
| 背景 | **透明（PNG alpha）** —— 图标/兵种/建筑/资源/野怪/UI 图标全部透明底 |
| 例外（不透明） | `ui_bg_parchment`、`map_tile_*`、`ui_btn` 是背景/贴图，**满幅、可平铺、不留透明**（见 §3.7） |
| 构图 | 主体居中、留边；**同类别统一相机角度与占画比**（成套关键） |
| 相机 | 资源图标=正面平视；建筑/资源田=**3/4 俯视**；兵种/野怪=3/4 正面；UI 图标=正面符号 |
| 禁止 | 文字、水印、边框、外投影、写实照片感、现代物件 |
| 命名 | 见每行「文件名」，全小写下划线，覆盖同名占位图即可 |

## 四、成套一致性检查（出图后自查）

- [ ] 同类别相机角度一致（10 建筑都 3/4 俯视、10 兵种都 3/4 正面）
- [ ] 同类别占画比一致（主体大小、留边一致，不忽大忽小）
- [ ] 透明底该透的透了；`ui_bg`/`map_tile_*` 满幅不透明且能平铺
- [ ] 无文字/水印/边框/外投影
- [ ] 缩到 128×128 后仍能一眼辨认（剪影清晰）
- [ ] 文件名与 §三完全一致 → 覆盖 `packages/client/public/art/` → `npm run build`
