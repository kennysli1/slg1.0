# 部署手册 · 腾讯云轻量应用服务器（实操版）

> 目标：把游戏部署到公网，你和朋友能通过 `http://你的公网IP:8080` 一起玩。
> 系统：Linux。数据用 JSON 文件持久化（无需装数据库）。进程用 pm2 保活。
> 标 **【你做】** 的需你在服务器/本地操作；命令可直接复制。遇到任何报错把屏幕内容发我。

---

## 步骤 0 · 先准备（控制台操作）【你做】

1. 登录腾讯云控制台 → 轻量应用服务器 → 你的服务器。
2. 记下 **公网 IP**：`____________`
3. 「防火墙」标签 → 添加规则：放行 **TCP 8080**（应用端口）。SSH 的 22 通常已开。
4. 用「登录」按钮打开网页终端，敲 `cat /etc/os-release`，看是 Ubuntu/Debian 还是 CentOS。

---

## 步骤 1 · 服务器装 Node.js 20【你做，在服务器终端】

**Ubuntu / Debian：**
```bash
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt-get install -y nodejs
node -v        # 应显示 v20.x
```

**CentOS / 腾讯云 OpenCloudOS：**
```bash
curl -fsSL https://rpm.nodesource.com/setup_20.x | sudo bash -
sudo yum install -y nodejs
node -v
```

装 pm2（进程守护，开机自启+崩溃重启）：
```bash
sudo npm install -g pm2
```

---

## 步骤 2 · 把代码弄到服务器

代码在你本地 `e:/slg1.0`。三选一，**推荐 A（用 Git）**：

### 方式 A：通过 Git（推荐，以后更新方便）
先把代码推到 Gitee/GitHub（私有仓库即可），然后服务器上：
```bash
cd ~
git clone <你的仓库地址> kow
cd kow
```
> 还没建仓库的话告诉我，我给你建 git 仓库 + 推送的命令。

### 方式 B：直接打包上传
本地（你的电脑）打包——**排除 node_modules 和 data**：
```bash
# 在 e:/slg1.0 目录，用 git bash 或 WSL
tar --exclude=node_modules --exclude=data --exclude=dist -czf kow.tar.gz .
```
然后用腾讯云网页终端的「上传文件」把 `kow.tar.gz` 传上去，服务器上：
```bash
mkdir -p ~/kow && tar -xzf ~/kow.tar.gz -C ~/kow
cd ~/kow
```

---

## 步骤 3 · 安装依赖 + 构建前端【你做，在服务器 ~/kow】

```bash
npm install              # 装依赖（含 tsx）
npm run build            # 构建 shared + server + 前端静态文件
```
> `npm run build` 完成后，前端在 `packages/client/dist`，后端编译产物在 `packages/server/dist`，后端启动时会自动托管前端静态文件。

---

## 步骤 4 · 启动并保活【你做】

```bash
pm2 start ecosystem.config.cjs     # 按配置启动（端口8080，数据存 ./data）
pm2 save                           # 保存进程列表
pm2 startup                        # 按提示再执行它输出的一条命令 → 开机自启
```
查看状态/日志：
```bash
pm2 status                         # 看是否 online
pm2 logs kow                  # 看实时日志（Ctrl+C 退出查看）
```

## 步骤 5 · 验证【你做】

浏览器打开：`http://你的公网IP:8080`
- 看到登录页 → 成功！注册账号、选种族、进游戏。
- 把这个地址发给朋友，各自注册不同账号，就能在地图上看到对方、互相进攻。

## 日常运维速查

| 操作 | 命令 |
|------|------|
| 看运行状态 | `pm2 status` |
| 看日志 | `pm2 logs kow` |
| 重启（改了配置后） | `pm2 restart kow` |
| 停止 | `pm2 stop kow` |
| 更新代码后重新部署 | `git pull`（方式A）→ `npm install` → `npm run build` → `pm2 restart kow` |
| 改游戏数值 | 编辑 `config/*.csv` → `pm2 restart kow` |

> **存档在哪**：`~/kow/data/game.json`。备份它就是备份整个游戏世界。重启服务不丢数据。

## 安全提醒（上线须知）

- 当前用 HTTP（非加密）。和朋友内部玩够用；若要正式公开，建议后续加域名 + HTTPS（我可以帮你配 Nginx + 免费证书）。
- 密码已加密存储（scrypt），但传输是明文 HTTP——同理，正式公开需上 HTTPS。
- 防火墙只开了 8080 和 22，其它端口默认关闭，是安全的。
