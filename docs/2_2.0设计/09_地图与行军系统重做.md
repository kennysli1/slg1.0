# 09 · 地图与行军系统重做（六边形 + 真实路径 + 途中相遇）

> 状态：**已实现（阶段一）**。本文是"移动模型"这条线的唯一设计入口。
> 战斗结算是另一条线，见 `08_战斗系统重做设计.md`。两条线在"到达/相遇→交战"处对接。

---

## 一、总目标（一句话）

把地图从方格改成**六边形网格**，行军从"瞬时到达"改成**沿真实路径逐格推进**，
玩家能在地图上看到自己部队的**行军路径与实时位置**，敌对出征军**在途同格相遇即战**。

---

## 二、坐标系：axial (q,r)

- 全地图统一用六边形**轴坐标** (q,r)（redblobgames pointy-top 标准）。
- 几何唯一来源：`packages/server/src/infra/hex.ts`——`hexDistance` / `linePath`（逐格路径）/ `neighbors` / `hexKey`。
  内部转 cube 做距离与线性插值取整，保证路径每步都是相邻格。
- 存档、CSV、前端全部改 (q,r)：
  - `config/pve_spawns.csv` 列头 `x,y → q,r`（值语义变轴坐标）。
  - `player` / `pve` / `world_tile` 的坐标字段 `x,y → q,r`；地块 key `"x,y" → "q,r"`。
  - `world_meta.size` 语义从"方形半边长"变为"**六边形半径（环数）**"。
- 前端渲染：`packages/client/src/shared/utils/hex.ts` 负责 axial↔像素（pointy-top）、六边形顶点、插值。

> **存档不兼容**：坐标语义变了，升级需刷档（`npm run reset:season`）。

## 三、行军模型：逐格推进（每格一个 Scheduler tick）

`modules/movement.ts` 的 `Movement` 记录新增：
`path`(逐格路径) / `stepIndex` / `pos`(当前格) / `perStepMs` / `nextStepAt` / `status` / `stepToken`。

- **出征**：用 `linePath(from,to)` 算全程逐格路径；全程秒数 = `hexDistance / 最慢兵种速度(格·h⁻¹)`，
  平摊到每格得 `perStepMs`；登记首个 `step` 任务（铁律#3，时间统一走 Scheduler）。
- **step()**：前进一格 → 更新 `pos/nextStepAt` → 相遇检测 → 到终点则到达，否则登记下一格。
- **stepToken（步进令牌）**：每次登记"下一格"任务时记录令牌，回调携带登记时令牌，
  只有匹配才执行。用于作废因**相遇暂停**而遗留的过期定时任务，防止重复推进（并发安全关键）。
- **resume()**：重启后按 `pos/nextStepAt` 为 marching 的部队续跑（作废旧令牌）。
- **list()** 下发 `path/pos/stepIndex/status/perStepMs/nextStepAt`，前端据此画路径折线并在两格间插值动画。

## 四、途中相遇（本期规则）

- **谁交战**：仅**两支敌对出征军**（type=raid/attack）走到**同一格**（pos 相同）即战。
  返程军（return）视为脱战，**免疫相遇**。同一玩家的部队互不交战。
- **相遇后**：双方**原地暂停**（status=paused，作废各自遗留 step 任务）→ 结算 →
  **胜方按对方相对强度损失部分兵后继续原定行军**，**败方全灭消失**。
- **战报**：发 `movement.Intercepted` Event（胜/败双方各一份，带相遇格 `at`、幸存兵力）。

### 与战斗系统（doc 08）的对接

- **到达目标格**：不自己结算，发 `combat.Engage` 交给有状态战斗模块开/并入战场；
  战斗结束 Combat 发 `combat.BattleEnded`，movement 监听后为幸存者安排带战利品返程。
  坐标对 Combat 为**不透明透传**（字段名 `fromXY/toXY/targetXY` 沿用，值是 `{q,r}`）。
- **途中相遇结算**：本期用 movement 内**自包含的一次性强弱结算占位**（`resolveFieldEncounter`），
  不依赖尚未就绪的"野战有状态战斗"。
  **TODO(阶段二)**：改为发"野战 `combat.Engage`"交给逐 tick 战斗；相遇期间双方 paused，
  `BattleEnded` 后由 `onBattleEnded` 恢复行军。届时删除占位结算。

## 五、前端可视化

- `features/map/map.ts` 从 `<div>` 方格网格改为 **SVG 六边形网格**：
  视野内每格一个 `<polygon>`（本城/玩家村/野怪着色），承载图标。
- **行军路径**：自己每支部队画一条 `polyline` 折线（返程用不同色）。
- **实时位置**：每支部队一个 `<g>` 标记（⚔️/🏇/🏠），用 `requestAnimationFrame` 按
  `nextStepAt/perStepMs` 在当前格与下一格间**像素插值**平滑移动；paused 变色。
- 点击目标格 → 目标面板（坐标显示 q,r，距离用 `hexDistance`）→ 出征沿用 SendRaid/SendAttack。

## 七、开放问题 / 阶段二

1. **野战相遇接入有状态战斗**：删除 `resolveFieldEncounter` 占位，改发野战 `combat.Engage`。
2. **相遇的并入语义**：目前一格只处理"两支"相遇；多支同格如何合并阵营（同盟/混战）待定。
3. **调参**：`perStepMs` 由 speed 决定，六边形距离比方格欧氏略大，需重估速度手感。
4. **视野与雾**：目前 GetArea 拉半径 25 全量下发，后续可做视野/雾遮蔽。
