# 11 · 建筑系统重做（三区结构 + 城镇中心解锁槽位 + 配置驱动）

> 状态：**策划定稿（2026-07-03，用户确认骨架）**，待进入架构与实现。
> 本文是"建筑系统"这条线的唯一设计入口。它替换旧的扁平槽位模型（`buildings.csv` + `fields.csv` 各自一摊、槽位写死）。
> 与之对接：资源惰性结算（经济）、兵种量产（军事）、二期的掠夺/攻城（战斗）。

---

## 一、总目标（一句话）

把建筑从"一堆扁平槽位照攻略铺满"改成**三区结构**：一栋**城镇中心**作为唯一的成长阀门，随它升级**分别解锁城内、城外两个独立槽位池**；玩家点击空槽 → 侧边栏列出该池可建建筑 → 自由选择建什么。**槽位总量永远小于想建的建筑总数**，逼出"每局都要做有意义取舍"的乐趣。

---

## 二、三区结构

| 区 | 定位 | 槽位来源 | 放什么 |
|---|---|---|---|
| **城镇中心** center | 中央地块，不占槽 | —（它就是阀门本身） | 只有它自己；升级它 = 解锁下面两池的槽位 |
| **城内区** inner（城墙内） | 民生 / 研发 / 存储 | 城镇中心等级 → `innerSlots` | 仓库、粮仓、学院、铁匠铺、市场、城墙 |
| **城外区** outer（前沿田野） | 生产 / 量产军队 | 城镇中心等级 → `outerSlots` | 资源田、产量强化建筑、兵营/马厩/工坊/校场、集结点 |

### 主题内核：研发在内、量产在外

- 兵种的**研发在城内**（学院解锁兵种、铁匠铺升攻防）——安全区，代表你"积累的科技成果"。
- 兵种的**量产在城外**（兵营/马厩/工坊）——前沿，代表你"当前的产能"。

首版这条分工是**纯风味**；二期加入暴露风险后立刻变成博弈——敌人能烧你城外兵营拖慢出兵，但碰不到城内的科技成果。

### 两池隔离（首版城内外区分意义的来源）

首版不做暴露风险（用户定），因此城内/城外的区分**不靠风险，靠槽位池隔离**：
- 城外的地**不能**盖仓库/学院；城内的地**不能**种田/盖兵营。
- 两池各自由城镇中心等级独立解锁 → "这一级我优先解锁的城内槽给仓储还是科技""城外槽给经济还是军队"本身就是取舍。
- 这套空间分区也为二期风险机制预留了现成结构（城外池 = 暴露区）。

## 三、城镇中心 = 唯一成长节奏阀门

每升 1 级城镇中心 → "叮"，按配置表开放城内/城外槽位，并可作为高级建筑的前置里程碑。这是全局唯一的成长仪表盘，也是科技树的根。

**曲线草案（数值待调，最终以配置表为准）：**

```
城镇中心Lv :  城内槽  城外槽   里程碑
   1        :   2      3      开局
   3        :   3      5      解锁「学院」入口
   5        :   4      7      解锁「马厩」「产量强化建筑」
   8        :   6     10      解锁「工坊(攻城器械)」
  12        :   8     14
  20(满)    :  10     18
```

**设计红线**：总槽位**明显小于**"想盖的建筑总数"（用户定：**硬取舍/大差值**）。开局城内 6 类建筑却只给 2 槽 → 立刻要选"仓储 / 科技 / 城墙"；满级也盖不下所有想要的东西，玩家必须**放弃整条线**（如"不盖马厩就彻底没有骑兵"）。这是把"照攻略铺满"变成"每局必须选流派"的核心机制，也天然造就 PvP 的流派差异化。

> **硬取舍的数值含义**：满级城内 10 槽 vs 城内 6~8 类可升建筑（含高级建筑还会更多）；满级城外 18 槽，但 4 类资源田若各铺多块（如各 3~4 块）就吃掉 12~16 槽，留给产量强化 + 全套军事建筑的槽位严重不足 → "经济流铺满资源田放弃军事" vs "军事流少铺田靠掠夺补给"的真实分歧。具体差值配合数值验证。

## 四、建筑清单（城内 / 城外分池）

### 城外 outer（生产 + 量产）
- **资源田**：伐木场、采泥场、铁矿、农田——**多块同类田、逐格升级**。稀缺感由 `outerSlots` 总量控制：铺满多块资源田就没剩几个外槽给军事/强化。
- **产量强化**：锯木厂 / 砖窑 / 熔炉 / 磨坊（给对应资源 +%）——**和军事建筑抢城外槽**，构成城外核心取舍："外槽给经济还是给军队"。
- **军事量产**：兵营、马厩、工坊、校场、集结点。

### 城内 inner（民生 + 研发）
- 仓库（木/泥/铁上限）、粮仓（粮上限）
- 学院（研究解锁兵种）、铁匠铺（升兵种攻防）
- 城墙（防御加成；二期→保护半径）、市场（后置，先占位）

## 五、"完全自由"靠数值软约束防 all-in（不设硬性建筑数量上限）

用户定：槽位内建什么完全自由，不设"最多几个某建筑"的硬限制。防极端 all-in 全靠数值软约束：

1. **四资源联动**：任何建造/升级吃 4 种资源，偏科种木材 → 泥铁粮卡死自己。
2. **crop 闸门**：兵吃粮、建筑有小额粮耗，净产为负则衰减（见系统清单 A4，弱化惩罚）。
3. **仓储上限**：不盖仓库就溢出浪费 → 逼你留城内槽给仓储。
4. **ROI 递减**：造价增长率略高于产量增长率，高级性价比下降，抑制无脑堆单一建筑。

## 六、配置驱动（本次重构的核心工程原则）

新增/调整建筑与槽位**只改 CSV**，不改代码。两处关键改动：

### ① `buildings.csv` 新增列 `zone`
建筑归属城内/城外/中心，只需在建筑表标一个字段。新增建筑时写明 zone 即可，符合"加内容只改配置"的需求。
```
zone 取值：center / inner / outer
```
- **资源田合并进 `buildings.csv`（已定）**：`fields.csv` 废弃，资源田统一为 `zone=outer` 的建筑，用产出字段（`resource` / `prodBase` / `prodGrowth`）区分。这样"点空槽 → 侧边栏 → 建造"是全城唯一逻辑，侧边栏不用兼容两张表。非资源田这些产出字段留空即可。

### ② 新增槽位表 `town_center_slots.csv`
城镇中心每级开放多少城内/城外槽 + 里程碑说明。
```
tcLevel, innerSlots, outerSlots, unlockNote
1,        2,          3,          开局
3,        3,          5,          解锁学院
5,        4,          7,          解锁马厩/产量强化
8,        6,          10,         解锁工坊
12,       8,          14,
20,       10,         18,         满级
```
- 槽位数量随等级开放的配置化，是用户明确要求（"配在表里，比如1级开放2内城3外城"）。
- 中间未列出的等级用线性/就近插值或逐级补全（实现时定，倾向表里逐级写全，避免歧义）。

### ③ 前置依赖 `requires` 继续用建筑ID:等级
高级建筑要求城镇中心达 X 级，直接用现有 `requires` 机制（如 `main:5`）。

## 七、建造交互

- **点城外/城内空槽** → 右侧（或底部）滑出侧边栏，列出**该池当前可建建筑**：图标 / 名称 / 造价 / 耗时 / 前置。
  - 前置不满足的建筑**灰显**并写明缺什么（如"需城镇中心 5 级"）。
- 选中 → 确认 → 进建造队列。
- **点已建建筑** → 详情面板（升级 / 信息 / 二期：拆除降级）。
- **建造队列**：开局 2 条；可设计成城镇中心升级再 +1 条（让 TC 更香）。
- 侧边栏只列**当前区（inner/outer）**的建筑，天然实现两池隔离。

## 八、二期预留（现在不做，结构留好）

- **暴露风险**：城外池 = 现成的暴露区。掠夺先破外围、攻城器械烧/降城外建筑。
- **城墙保护半径**：城墙升级 → 把一部分城外槽纳入保护。
- **前沿防御建筑**：前哨 / 瞭望塔 / 陷阱等，占城外槽。
- 这些都不改本文的三区骨架，只在城外区之上叠机制。

## 九、与旧模型的差异 & 迁移

| 旧 | 新 |
|---|---|
| `buildings.csv` + `fields.csv` 两张扁平表，槽位写死 | 统一建筑表 + `zone` 列；槽位由 `town_center_slots.csv` 按 TC 等级开放 |
| 城内外无机制区分 | 两池隔离（首版）→ 暴露风险（二期） |
| 主基地 main 降建造时间 | 升级为**城镇中心**：既降时间，又是槽位阀门 + 科技树根 |
| 资源田固定 18 格铺满 | 多块同类田逐格升级，但总量受 `outerSlots` 卡，产生取舍 |

> **存档不兼容**：建筑数据结构变了，升级需刷档（`npm run reset:season`，见部署规范）。
