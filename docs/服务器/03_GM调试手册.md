# GM 调试手册

> 适用对象：开发者 / 运维，用于查数据、改数据、排查 BUG。
> 两个工具：**GM HTTP API**（增删改查 Store）和 **结构化日志**（实时打印关键事件到文件）。

---

## 一、GM HTTP API

### 启停

服务器**默认启用**，随 `npm run dev:server` / `npm start` 一起启动，无需额外配置。

如需在生产环境关闭：

```
GM_ENABLED=off npm start
```

如需加 Token 保护（防止局域网内其他人随意访问）：

```
GM_TOKEN=mysecret npm start
# 之后每次请求都要加 Header：
# X-GM-Token: mysecret
```

---

### 端点一览

假设服务跑在 `http://localhost:8080`。

| 方法 | 路径 | 作用 |
|------|------|------|
| GET | `/gm/collections` | 列出所有集合名 + 文档数 |
| GET | `/gm/:collection` | 列出集合内所有文档（支持分页） |
| GET | `/gm/:collection/:key` | 读取一条文档 |
| PUT | `/gm/:collection/:key` | 写入/覆盖一条文档（body 为 JSON） |
| DELETE | `/gm/:collection/:key` | 删除一条文档 |
| DELETE | `/gm/:collection?confirm=yes` | 清空整个集合（需加 `?confirm=yes`） |

---

### 常用集合名

| 集合 | 内容 |
|------|------|
| `player` | 玩家账号（id → 玩家信息） |
| `player_byname` | 用户名 → playerId 的反查索引 |
| `player_byvillage` | villageId → playerId 的反查索引 |
| `economy` | 每村资源存量/产率/容量 |
| `building` | 每村建筑等级/队列 |
| `military` | 每村兵力/训练队列/铁匠等级 |
| `movement` | 在途行军记录 |
| `battle` | 进行中的战斗 |
| `pve` | PvE 目标守军/战利品/重生状态 |
| `world_tile` | 地图地块（村庄/PvE/空地） |
| `notifications` | 每村通知记录 |

---

### 示例：查数据

```bash
# 列出所有集合
curl http://localhost:8080/gm/collections

# 查某个集合的所有文档（默认返回前50条）
curl http://localhost:8080/gm/economy

# 分页
curl "http://localhost:8080/gm/economy?limit=20&offset=40"

# 查某个村庄的经济数据（key 通常是 villageId，如 v1）
curl http://localhost:8080/gm/economy/v1

# 查某个玩家账号（key 是 playerId，如 p1）
curl http://localhost:8080/gm/player/p1

# 查进行中的战斗
curl http://localhost:8080/gm/battle
```

---

### 示例：改数据

> **注意**：PUT 是全量覆盖，不是局部更新。建议先 GET 拿到完整文档，改字段后再 PUT 回去。

```bash
# 1. 先读出当前数据
curl http://localhost:8080/gm/economy/v1
# 返回示例：{"villageId":"v1","resources":{"wood":120,"clay":80,"iron":50,"crop":200},...}

# 2. 修改后写回（把 wood 改成 9999）
curl -X PUT \
  -H "Content-Type: application/json" \
  -d '{"villageId":"v1","resources":{"wood":9999,"clay":80,"iron":50,"crop":200},"baseRate":{"wood":0.138,...},"capacity":{"wood":800,...},"cropUpkeep":{},"settledAt":1000000}' \
  http://localhost:8080/gm/economy/v1
```

---

### 示例：删数据

```bash
# 删除一场进行中的战斗（慎用，不会自动发返程事件）
curl -X DELETE http://localhost:8080/gm/battle/bt-1

# 清空所有在途行军（刷档用，或卡死时应急）
curl -X DELETE "http://localhost:8080/gm/movement?confirm=yes"
```

## 二、结构化日志

### 日志位置

服务器**默认全开**，同时输出到：

- **控制台**（实时可见）
- **`data/logs/YYYY-MM-DD.log`**（按天滚动，自动清理7天前的旧文件）

---

### 控制开关

```bash
# 默认：全部模块日志都开（等同于不设置）
GAME_LOG=all npm run dev:server

# 只看战斗和经济日志
GAME_LOG=combat,economy npm run dev:server

# 关闭全部日志
GAME_LOG=off npm run dev:server
```

---

### 日志格式

```
[HH:mm:ss.mmm][模块名] 事件描述 {JSON数据}
```

示例：

### 常见排查场景

**Q：战斗结果不对（伤害/掠夺数量异常）**
1. `GAME_LOG=combat,economy` 启动，复现操作
2. 搜日志 `grep "战斗开始\|战斗结束\|掠夺" data/logs/$(date +%Y-%m-%d).log`
3. 看"战斗开始"里双方 `traits` 是否正确加载，"掠夺前"的 `lootable`/`totalCarry` 是否符合预期

**Q：兵力返程后没回来**
1. 搜 `grep "返程到达" data/logs/...log` 确认事件是否触发
2. 再查 `curl http://localhost:8080/gm/military/v1` 看兵力是否入账

**Q：想直接给玩家加资源测试**
```bash
# 读当前数据
curl http://localhost:8080/gm/economy/v1

# 修改 resources 字段后写回
curl -X PUT -H "Content-Type: application/json" -d '{...}' http://localhost:8080/gm/economy/v1
```

**Q：想清掉某场卡住的战斗**
```bash
curl http://localhost:8080/gm/battle          # 先看 battleId
curl -X DELETE http://localhost:8080/gm/battle/bt-1
```
