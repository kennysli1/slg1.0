## 一分钟速览

- **存哪**：单个 JSON 文件，默认 `data/game.json`（环境变量 `DATA_PATH` 可改）。整个 `data/` 被 git 忽略。
- **怎么存**：内存操作 + 防抖落盘（原子替换），进程退出前 flush。实现见 `packages/server/src/infra/store.ts`。
- **两类数据**：**账号**（用户名/密码/种族/地图位置）与**游戏进度**（资源/建筑/军队/在途部队/野怪/地图）分开管理，刷档时可选择只清进度、保留账号。
- **刷档**：三个命令，按改动性质选——

  | 命令 | 保留账号 | 保留地图位置 | 用在什么时候 |
  |------|:--:|:--:|------|
  | `npm run reset:season` | ✓ | ✓ | 新赛季重开：只清进度 |
  | `npm run reset:respawn` | ✓ | ✗ | 地图尺寸/布局也改了：重排位置 |
  | `npm run wipe:all` | ✗ | — | 彻底删档，回到零玩家 |

  三个命令执行前都会**自动备份**当前存档到 `data/backups/`。详见操作手册。

- **未来换数据库**：只改 `infra/store.ts` 一个文件（实现 `Store` 接口），领域模块零改动。见操作手册末节。
