# 01 · Travian 核心框架反向拆解

> 本文还原 Travian 的整体玩法骨架，是后续所有系统设计的根。

## 1. 一句话定义

Travian 是一款**浏览器/移动端、实时、多人、PvP 为核心的策略战争游戏（SLG）**。
玩家从一座村庄起步，发展经济 → 训练军队 → 扩张领土 → 加入联盟 → 在赛季终局争夺"世界奇迹"获得全服胜利。

核心特征：
- **实时持续运行**：建造、行军、训练都按真实时间计时，即使离线也在推进。
- **赛季制**：单局（一个服务器）通常持续约 200–300 天，有明确的胜利条件与结束，然后开新一轮。
- **强 PvP + 强社交**：单人无法获胜，必须依赖联盟协作。

## 2. 核心玩法循环（Game Loop）

```
                ┌──────────────────────────────────────────┐
                │                                            │
                ▼                                            │
   采集/产出资源 → 建造与升级建筑 → 训练军队 → 扩张(建新村)    │
        │              │              │            │         │
        │              │              │            ▼         │
        │              │              │      掠夺/进攻他人 ───┤
        │              │              │            │         │
        │              ▼              ▼            ▼         │
        └──────→ 防守/被掠夺 ←── 联盟协作/外交 ←── 积累实力 ──┘
                                       │
                                       ▼
                              终局：世界奇迹竞赛 → 全服胜利
```

**短循环（分钟~小时级）**：看资源 → 排建造队列 → 发掠夺/行军 → 收菜。
**中循环（天级）**：升级关键建筑、攒文化点开新村、刷英雄冒险、参与联盟战。
**长循环（赛季级）**：抢神器 → 备战终局 → 建造世界奇迹 → 决出胜负。

## 3. 五大系统支柱

| 支柱 | 内容 | 对应文档 |
|------|------|----------|
| **经济系统** | 4 种资源、资源田、22 个建筑槽、仓库/粮仓、市场交易 | `02` |
| **军事系统** | 5 部族 × 约10兵种、攻防/战斗公式、攻城、英雄、掠夺 | `03` |
| **扩张系统** | 文化点(Culture Points)、绿洲、建立/占领村庄、人口 | `02` |
| **社交系统** | 联盟(≤60人)、外交、共享情报、聊天论坛 | `04` |
| **终局系统** | Natars、神器(Artifacts)、世界奇迹、赛季胜利与重置 | `04` |

## 4. 部族（Tribes）

原版可选 5 个部族，各有鲜明定位（详细兵种见 `03`）：

| 部族 | 定位 | 特色 |
|------|------|------|
| **罗马人 Romans** | 全能/后期强 | 可同时建造资源田与建筑；兵种贵但强；耗铁 |
| **高卢人 Gauls** | 防守/速度 | 防守强、移动快、陷阱；新手友好；耗泥 |
| **条顿人 Teutons** | 进攻/掠夺 | 兵便宜、掠夺强、地窖大；早期压制力强；耗木 |
| **匈奴人 Huns** | 骑兵/机动 | 骑兵流，可在地图上设营 |
| **埃及人 Egyptians** | 防守/经济 | 防守强、资源加成、水利建筑 |

> 部族是 Travian 差异化与重开局动机的核心，2.0 是否保留/扩充是关键决策点（见 `06`）。

## 5. 关键设计哲学（值得 2.0 继承或挑战）

1. **时间即资源**：一切需要等待，"加速"是付费(Gold)与策略的核心。
2. **资源是硬约束**：所有成长都被 4 种资源 + 人口(crop)限制，形成经济博弈。
3. **不可单人通关**：胜利绑定联盟与全服协作，制造社交粘性。
4. **掠夺驱动 PvP**：弱者被强者掠夺，刺激持续上线与防守投入。
5. **赛季重置**：定期清零保证公平与新鲜感，但也意味着玩家流失风险。

## 6. 平台与技术形态（原版）

- 形态：浏览器网页 + 移动 App，**服务器权威**的回合外实时模拟。
- 后端：以定时事件队列驱动（建造完成、行军到达、训练完成等都是定时任务）。
- 前端：以"村庄视图 / 资源视图 / 地图视图 / 报告"为主的信息展示界面，交互轻。

> 技术形态决定了 2.0 的架构选型，详见 `07_开发计划.md`。
